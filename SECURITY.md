# Security Policy

## Supported Versions

Only the latest release line is considered supported for security fixes.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Use one of the following private channels when the public repository is available:

- GitHub Security Advisories private reporting
- direct contact to the maintainer account `@JunggiKim`

When reporting a vulnerability, include:

- affected version
- impact summary
- reproduction steps or proof of concept
- any proposed mitigation

The project aims to acknowledge reports promptly and coordinate a fix before public disclosure.
